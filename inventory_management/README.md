# Inventory Management

A small inventory project with a Flask API backed by SQLite and an optional lightweight web GUI.

By default, data is stored in `data/inventory.db`

## API

Most successful item read/write responses use an `item` object or an `items` list. Delete responses return the deleted item id and a status message.

#### Data Model

Items have:

- `id`: integer, generated by SQLite
- `object_name`: required non-empty string
- `qnty`: required integer greater than or equal to 0
- `location`: optional string
- `category_tags`: optional list of strings

#### Response

Single-item responses:

```json
{
  "item": {
    "id":1,
    "object_name":"name of item",
    "qnty":1,
    "location":"location of item if set",
    "category_tags":["list","of","tags","if set"]
  }
}
```

List and search responses:

```json
{
  "items": [
    {
      "id":1,
      "object_name":"name of item",
      "qnty":1,
      "location":"location of item if set",
      "category_tags":["list","of","tags","if set"]
    }
  ],
  "count": 1
}
```

Delete responses:

```json
{
  "message": "deleted",
  "id": 1
}
```

Error responses:

```json
{
  "error": {
    "code": "bad_request",
    "message": "An error message",
    "details": ["Optional validation details"]
  }
}
```

Current API error codes include `bad_request`, `not_found`, `method_not_allowed`, and `internal_error`. The optional GUI proxy can also return `api_unavailable`.

#### Endpoints
##### GET
<details>
  <summary>/items (Get all items)</summary>

  ```sh
  # Example
  curl -s http://localhost:5000/items
  ```
</details>

<details>
  <summary>/items/&lt;ID&gt; (Get item by id)</summary>

  ```sh
  # Example
  curl -s http://localhost:5000/items/1
  ```
</details>

<details>
  <summary>/items?q= (Fuzzy search items)</summary>

**Search matches item name, location, and tags.**
  ```sh
  # Example
  curl -s http://localhost:5000/items?q=batteries
  ```
**Use `*` to return all items (same as /items)**
  ```sh
  # Example
  curl -s http://localhost:5000/items?q=*
  ```
</details>

##### POST
<details>
  <summary>/items (Add item)</summary>

  ```sh
  # Example: Add 'AA batteries'
  curl -s -X POST http://localhost:5000/items \
    -H 'Content-Type: application/json' \
    -d '{"object_name":"AA Batteries","qnty":8,"location":"Utility drawer","category_tags":["batteries","small"]}'
  ```
</details>

##### PUT
<details>
  <summary>/items/&lt;ID&gt; (Update by ID)</summary>

  ```sh
  # Example: Update quantity of ID=1
  curl -s -X PUT http://127.0.0.1:5000/items/1 \
    -H 'Content-Type: application/json' \
    -d '{"qnty":10}'
  ```
</details>

##### DELETE
<details>
  <summary>/items/&lt;ID&gt; (Delete item by ID)</summary>

  ```sh
  curl -s -X DELETE http://127.0.0.1:5000/items/1
  ```
</details>

## Running

### &ensp;&ensp;Docker

Run with Docker Compose (default just starts the API):

```sh
docker compose up --build api
```

Run the API plus GUI:

```sh
docker compose --profile gui up --build
```


### &ensp;&ensp;Python

Install dependencies:

```sh
python -m pip install -r requirements.txt
```

Start the API (default `http://localhost:5000`):

```sh
python api.py
```

Start the optional GUI in a second terminal (default `http://localhost:5001`):

```sh
API_URL=http://127.0.0.1:5000 python web.py
```

Run the API tests:

```sh
python -m unittest discover
```

*NOTE: Tests use a temporary SQLite database and do not touch `data/inventory.db`.*
